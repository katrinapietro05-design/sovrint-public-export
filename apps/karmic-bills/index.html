Purpose

What a karmic bill is

A governed record of an unresolved action, obligation, exchange, harm, promise, responsibility, or repair process. It can hold monetary and non-monetary value simultaneously.

Operating rule

Closure requires evidence

A record is not closed because attention moved elsewhere. Closure requires explicit criteria, a closure statement, a date, and supporting evidence or acknowledgement.

Status model

Lifecycle

Draft → Open → Acknowledged → In Repair → Fulfilled → Closed. Disputed and Archived remain governed alternate states, not deletion pathways.

Data custody

Local-first persistence

This release stores records in the browser for immediate portability. Export regularly. A server-backed adapter can replace local storage when integrated into the Sovereign Systems Hub.
